# tetra-server

`TETLABO` のサーバーサイド実装です．  
オンラインでマルチプレイをやる際の基盤として使用することを想定しています．

# ローカル環境構築

Rustをインストールしている状態で

```bash
$ cargo run
```

で実行できます．

または，フロントエンド実装 `tetra-programming` が同ディレクトリにある状態なら， `tetra-programming` の開発サーバー起動コマンドで同時に起動できます．

# 実装

通信は全てクライアントサーバーモデルで， WebRTC を使用して通信します．  
シグナリングには WebSocket を使用します．

全て C/S モデルのため，クライアント同士の P2P 通信は行われず，サーバーが通信を経由する形になります．

# Environment Variables

| KEY                 | 説明                                                                                  | 例               |
| ------------------- | ------------------------------------------------------------------------------------- | ---------------- |
| `HTTP_ADDR`         | HTTPサーバーをホストするアドレスを指定します                                          | `127.0.0.1:8080` |
| `ALLOW_VERSIONS`    | クライアントの許可するバージョンをカンマ区切りで指定します                            | `2.0.0,1.3.0`    |
| `CF_TURN_TOKEN_ID`  | Cloudflare TURN のトークンIDを指定します (指定されない場合は，TURNは利用されません．) | `xxxx`           |
| `CF_TURN_API_TOKEN` | Cloudflare TURN の APIトークンを指定します                                            | `xxxx`           |
